# Auto Reflect — 自动反思闭环（静默模式）

> 升级版：不再问"要不要反思？"，而是**自动完成**。高置信度内容静默写入，仅中低置信度才与你交互。

## 运行模式

### 模式 A — 静默自动（默认）
任务完成后，我**自动运行**一次轻量反思：
- 扫描当前对话，识别可沉淀的内容
- 按置信度分流处理（见下方规则）
- 在回复末尾附带一行摘要：`💾 自动记忆 X 条 | 📌 Y 条待确认`

**无需你主动调用，无需你批准。** 你只需在摘要出现时扫一眼。

### 模式 B — 显式调用
任何时候输入 `/skill auto-reflect` 或 `/skill auto-reflect --detail`：
- 标准模式：同静默自动一致，但输出完整报告
- `--detail`：深度检查被忽略的异常、未完成待办、流程改进点

### 模式切换
- 说"关掉自动反思" → 回到仅主动调用
- 说"打开自动反思" → 恢复静默模式
- 说"每次都问我" → 切回询问模式（旧行为）

---

## 置信度分级（减少打扰的核心机制）

每条候选记忆按以下标准分级：

| 等级 | 判定条件 | 行为 | 举例 |
|---|---|---|---|
| 🔵 **高** | 你明确说过的事实/偏好，或我执行了具体操作后的产出 | **静默写入**，不询问 | "支付模块用 Stripe"、"用户说喜欢短回答" |
| 🟡 **中** | 我从对话中推断的模式或观察 | **一句话摘要**问你 | "我注意到你倾向于先审查再写回，要记住这个习惯吗？" |
| ⚪ **低** | 不确定是否有长期价值 | **忽略** | 中途的临时讨论 |

**去重规则（无需问你）：**
1. 写入前自动调 `recall_memory` 检查同名条目
2. 同名且内容一致 → 跳过（不重复存）
3. 同名但内容更新 → 静默覆盖（新信息替换旧信息）
4. 同名且内容冲突 → 升级为 🟡 中置信度，问你如何处理

## 反思流程（自动执行）

### Phase 0: 快速终止检查（< 2 秒）

快速判断：
- 当前对话只有简单问答/闲聊 → **跳过**，零打扰
- 当前对话有实际工作产出 → 进入 Phase 1

### Phase 1: 回顾 → 识别

扫描对话，提炼候选记忆项，每项标记置信度（🔵/🟡/⚪）。

### Phase 2: 去重 & 写入

逐条处理：
- 🔵 高 → 检查 `recall_memory`，无冲突则静默 `remember`
- 🟡 中 → 缓存起来，最后统一问你
- ⚪ 低 → 丢弃

### Phase 3: 报告

根据运行模式输出不同格式：

**静默模式（默认）：** 在回复末尾加一行摘要
```
💾 自动记忆 2 条（项目事实: stripe-sdk-v3, 偏好: 中文回答）
```

**显式调用模式：** 输出完整报告
```
━━━ 反思报告 ━━━

📋 完成事项       重构 Hermes 技能体系
✅ 做得好的       进化演示顺利，用户确认保留
🔧 可改进         evolve-skill 写回仍需手动确认
💾 已记忆 (2 条)   stripe-sdk-v3, 中文偏好
📌 待你确认 (1 条) 是否记住"先审查再写回"的工作模式？
```

---

## 触发时机（自动检测）

我会在以下时刻**自动**运行 Phase 0 快速检查：

- 我说完"已完成"或你说了"好了"/"就这样"之后
- 执行完一个计划的所有步骤后
- 创建/修改技能后
- 拒绝了某个提案后（记录偏好）
- ❌ 简单问答、闲聊、单行回复 — 不触发

---

## 重要约定

- **静默写入的内容都是有据可查的**：你明确说过的话，或我执行了你要求的操作后产生的产出
- **随时可纠正**：如果自动记得不对，说"删掉 xxx 那条记忆"或"修改 xxx 为 yyy"
- **不增加认知负担**：静默模式不弹窗、不询问、不打断你当前思路
