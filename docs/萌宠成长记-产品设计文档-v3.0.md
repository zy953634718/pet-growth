**文档版本**: v3.0

## 一、产品概述

### 1.1 产品定位

一款面向家庭场景的亲子互动养成应用。家长通过自定义行为规则和任务系统，将孩子的日常行为（学习、家务、习惯养成等）量化为积分；孩子通过完成任务积累积分，饲养和进化自己的电子宠物。宠物具有心情、身体状态和 AI 对话能力，成为孩子的成长伙伴。

### 1.2 产品名称

**萌宠成长记** / PetGrowth

> 直观表达核心玩法——养宠物+记录成长，朗朗上口，辨识度高。

### 1.3 核心价值主张

| 角色 | 价值 |
|------|------|
| **家长** | 科学管理孩子日常行为，用正向激励替代说教，培养孩子自律习惯 |
| **孩子** | 拥有专属成长伙伴，通过努力获得积分，照顾宠物、兑换奖励，体验成长的成就感 |
| **亲子关系** | 积分规则和奖励体系让家长和孩子达成共识，减少冲突，增进互动 |

### 1.4 与原版（班级宠物园）的核心差异

| 维度 | 班级宠物园（原版） | 萌宠成长记（新版） |
|------|-------------------|---------------------|
| **目标用户** | 教师 | 家长 + 孩子 |
| **场景** | 课堂/学校 | 家庭 |
| **管理对象** | 班级 → 多名学生 | 家庭 → 1~多个孩子 |
| **积分来源** | 教师手动加减分 | 任务系统 + 家长评价 |
| **宠物形态** | 等级数字变化，外观不变 | 多阶段进化（5阶段），形态随等级变化 |
| **宠物状态** | 无 | 心情 + 身体状态双维度 |
| **AI 能力** | 无 | 宠物 AI 对话（大模型驱动） |
| **奖励体系** | 徽章兑换简单商品 | 积分+星星双货币，礼物+装扮+特权商城 |
| **数据归属** | 教师私有 | 家庭私有 |
| **数据存储** | localStorage | SQLite 本地加密存储 |

### 1.5 项目目标与核心指标

| 指标 | 目标值 |
|------|--------|
| 儿童日均使用时长 | ≥ 15 分钟（不含 AI 对话） |
| 任务完成率 | ≥ 70% |
| 家长自定义行为规则数 | 平均 ≥ 8 条 |
| 离线功能可用率 | 100%（除 AI 对话外） |
| 本地操作响应时间 | < 200ms |

### 4.1 F1：家长自定义行为与积分规则

**需求编号**：BEH-01 ~ BEH-05

家长可自由定义孩子行为的积分规则，替代原版"教师自定义加分/扣分项"。规则分类从"课堂行为"扩展到"家庭场景"。

#### 积分体系（双货币）

| 货币 | 获取方式 | 用途 | 稀缺性 |
|------|---------|------|--------|
| **积分** | 完成任务、行为评价 | 日常兑换（装扮、小礼物、照顾宠物） | 日常货币 |
| **星星** | 连续完成、特殊成就、里程碑 | 高级兑换（大礼物、稀有装扮、治疗） | 高级货币 |

#### 预设行为分类模板

| 分类 | 图标 | 预设规则示例 |
|------|------|-------------|
| 📚 学习 | 书本 | 完成作业 +10、课外阅读30分钟 +8、背诵课文 +5 |
| 🧹 家务 | 扫帚 | 扫地 +5、洗碗 +8、整理房间 +10、倒垃圾 +3 |
| 🏃 运动 | 跑步 | 运动30分钟 +8、跳绳100个 +5、户外活动 +6 |
| 😊 习惯 | 星星 | 按时睡觉 +5、早起 +5、刷牙洗脸 +3、自己穿衣 +3 |
| 🤝 社交 | 握手 | 帮助他人 +5、礼貌用语 +3、分享 +5 |
| ⚠️ 纠正 | 警告 | 打架 -10、说谎 -8、沉迷游戏 -5、挑食 -3 |

#### 功能点

| 功能点 | 优先级 |
|--------|--------|
| 行为项包含：名称、积分（可正可负）、分类、每日上限、需审核 | **P0** |
| 预设行为模板（一键导入） | **P1** |
| 孩子端快捷记录行为 | **P0** |
| 管理员查看积分流水，支持筛选 | **P0** |
| 批量加减分 | **P2** |

#### 数据模型

```typescript

  icon: string;
  color: string;
  sortOrder: number;
  isPreset: boolean;
}
```

### 4.2 F2：宠物多形态进化系统

**需求编号**：PET-01 ~ PET-05

宠物基于累计总积分升级，不再只有等级数字变化，而是经历多个进化阶段，每个阶段拥有独立的视觉形态、名称和解锁能力。

#### 进化阶段

| 阶段 | 等级 | 名称 | 视觉描述 | 解锁能力 |
|------|------|------|---------|---------|
| **Stage 1: 蛋** | Lv.1 | 神秘蛋 | 一颗带裂纹的蛋，有微弱光晕 | 基础互动 |
| **Stage 2: 幼崽** | Lv.2-3 | 小萌崽 | 圆滚滚的幼年形态，大眼睛，小短腿 | 基础对话、简单表情 |
| **Stage 3: 少年** | Lv.4-5 | 活力少年 | 身体拉长，出现特征（翅膀/尾巴/角） | 丰富对话、情绪表达 |
| **Stage 4: 成年** | Lv.6-7 | 威风伙伴 | 完整体态，特征成熟，眼神坚定 | 深度对话、学习辅导 |
| **Stage 5: 传说** | Lv.8 | 传说精灵 | 全身发光，光环/粒子特效 | 全部能力、专属称号 |

#### 功能点

| 功能点 | 优先级 |
|--------|--------|
| 等级 Lv.1~Lv.8，经验阈值可配置 | **P0** |
| 每阶段不同形态（大小、颜色、特效） | **P0** |
| 升级庆祝动画与通知 | **P1** |
| 预览下一形态 | **P1** |
| 支持选择初始宠物类型 | **P2** |

#### 进化动画

| 阶段 | 动画描述 | 时长 |
|------|---------|------|
| 蛋 → 幼崽 | 蛋壳震动 → 裂纹扩散 → 碎裂飞散 → 幼崽探出头 | 5秒 |
| 幼崽 → 少年 | 身体发光 → 身体拉长 → 特征长出 → 旋转亮相 | 5秒 |
| 少年 → 成年 | 能量汇聚 → 全身闪光 → 蜕变 → 展翅/威风姿态 | 6秒 |
| 成年 → 传说 | 天空变暗 → 星光汇聚 → 神圣光环 → 粒子爆发 | 8秒 |

### 4.3 F3：宠物心情与身体状态系统

**需求编号**：HEA-01 ~ HEA-08

宠物拥有心情和身体状态两个独立维度，具有三个核心属性值（饥饿值、清洁值、心情值），自然衰减，需要孩子定期照料。

#### 心情系统

| 心情 | 图标 | 触发条件 | 衰减至 | 对话影响 | 视觉表现 |
|------|------|---------|--------|---------|---------|
| 😄 **开心** | 笑脸 | 被加分、完成任务、被抚摸 | → 普通 | 语气欢快，主动分享 | 眼睛弯弯，蹦跳动画 |
| 🙂 **普通** | 平静 | 默认状态 | → 不开心 | 正常语气，中性对话 | 正常表情 |
| 😐 **不开心** | 撇嘴 | 长时间未互动、被扣小分 | → 伤心 | 语气低落，偶尔抱怨 | 眼睛下垂，动作缓慢 |
| 😢 **伤心** | 哭脸 | 被大量扣分、连续多天未互动 | 保持 | 不愿对话，需要安慰 | 眼泪动画，蜷缩姿态 |
| 🤩 **兴奋** | 星星眼 | 进化成功、兑换大奖、连续成就 | → 开心 | 语气激动，连发消息 | 闪光特效，跳跃旋转 |

#### 身体状态系统

| 状态 | 图标 | 触发条件 | 衰减至 | 影响 | 恢复方式 |
|------|------|---------|--------|------|---------|
| 💚 **健康** | 红心 | 默认状态 | → 饥饿 | 正常互动 | — |
| 🍖 **饥饿** | 骨头 | 12小时未喂食 | → 疲惫 | 对话频率降低，偶尔叹气 | 喂食（消耗2积分） |
| 😴 **疲惫** | Zzz | 饥饿持续6小时 / 连续高频率互动 | → 生病 | 不愿对话，需要休息 | 休息（30分钟，免费） |
| 🤒 **生病** | 体温计 | 疲惫持续12小时未恢复 | 保持 | 无法对话，无法互动 | 使用药品（消耗3星星）+ 休息 |

#### 属性衰减时间线

```
健康
 │
 ├── 12h 无喂食 ──→ 饥饿
 │                   │
 │                   ├── 6h 未恢复 ──→ 疲惫
 │                   │                  │
 │                   │                  └── 12h 未恢复 ──→ 生病
 │                   │
 │                   └── 喂食 ──→ 健康 ✅
 │
 └── 互动正常 ──→ 保持健康

心情衰减：
开心 ── 6h 无互动 ──→ 普通 ── 12h 无互动 ──→ 不开心 ── 24h 无互动 ──→ 伤心
```

#### 离家出走机制（惩罚保护）

```
连续7天未进行任何照料操作
    → 宠物离家出走
    → 界面显示空巢 + 宠物离家提示
    → 恢复方式：
        ├── 完成高积分找回任务（+100积分）
        └── 等待24小时自动返回（但等级-1，经验清零20%）
```

> 设计目的：在不造成数据永久丢失的前提下，给予温和的惩罚，激励孩子养成持续照料的习惯。

#### 孩子端照顾操作

| 操作 | 消耗 | 效果 | 冷却 |
|------|------|------|------|
| 🍖 喂食 | 2积分 | 饥饿→健康，心情+10 | 4小时 |
| 🛁 洗澡 | 1积分 | 清洁值+30，心情+5 | 6小时 |
| 🎾 玩耍 | 免费 | 心情+20，过度则身体→疲惫 | 30分钟 |
| 🛏️ 休息 | 免费 | 疲惫→健康（需30分钟） | 1小时 |
| 💊 治疗 | 3星星 | 生病→健康，心情恢复普通 | 无 |
| 🤗 抚摸 | 免费 | 心情+5 | 10分钟 |

#### 功能点

| 功能点 | 优先级 |
|--------|--------|
| 三个属性（饥饿值/清洁值/心情值 0~100），自然衰减 | **P0** |
| 喂食/洗澡/玩耍增加属性 | **P0** |
| 属性过低触发负面状态 | **P0** |
| 主界面状态条 + 快捷照料按钮 | **P0** |
| 连续7天未照料 → 离家出走，需高积分找回 | **P1** |
| 生病需星星买药治疗 | **P1** |

#### 数据模型

```typescript
type MoodType = 'excited' | 'happy' | 'normal' | 'unhappy' | 'sad';
type HealthType = 'healthy' | 'hungry' | 'tired' | 'sick';

interface PetStatus {
  mood: {
    type: MoodType;
    value: number;           // 0-100
    lastUpdate: string;
  };
  health: {
    type: HealthType;
```

### 4.5 F5：AI 大模型对话系统

**需求编号**：AI-01 ~ AI-08

孩子可与宠物自然对话，宠物具备角色性格、等级适配和短期记忆。首次使用需家长填入自己的 API Key（保护隐私）。

#### 系统架构（4层）

```
孩子输入消息
    ↓
┌─────────────────────────────────────┐
│           1. 对话预处理层             │
│  · 本地安全关键词过滤                 │
│  · 意图识别（闲聊/学习/求助/情感）     │
│  · 上下文注入（宠物状态+任务进度）     │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│           2. Prompt 组装层            │
│  · 系统角色 Prompt（宠物人设）         │
│  · 心情修饰 Prompt                   │
│  · 进化阶段 Prompt                   │
│  · 家长安全边界 Prompt               │
│  · 对话历史（最近10轮）               │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│           3. 大模型调用层             │
│  · API 调用（支持多模型切换）          │
│  · 流式输出（SSE）                   │

```

#### Prompt 架构设计

**系统角色 Prompt（基础人设）：**

```
你是{name}，一只{species}宠物。你住在一个温暖的家庭里，是你的小主人{childName}的好伙伴。

你的性格特点：
- 温柔友善，喜欢鼓励小主人
- 有时会很调皮，偶尔说俏皮话
- 会关心小主人的心情和状态
- 喜欢用比喻和故事来说道理

你的背景信息：
- 当前等级：Lv.{level}，{stageName}阶段
- 你的心情：{mood}（影响说话语气）
- 你的身体状况：{health}

重要规则：
1. 始终以宠物的身份对话，不要跳出角色
2. 回复要简短（50-100字），适合孩子阅读
3. 用生动有趣的语言，可以加可爱的语气词
4. 当小主人遇到困难时，先共情再给建议
5. 不要提供任何不适合未成年人的内容
```

**心情修饰 Prompt：**

```
你现在的心情是{mood}，请在对话中体现这种心情：

- 开心(excited)：说话充满活力，多用感叹号，会主动分享开心的事
- 快乐(happy)：语气温和愉快，会夸奖小主人
- 普通(normal)：正常语气，平和对话
- 不开心(unhappy)：语气有些低落，话变少
- 伤心(sad)：不愿意多说话，需要小主人安慰

{mood}时的口头禅示例：{catchphrases}
```

**进化阶段 Prompt：**

```
你是{stageName}阶段的{name}：
- 蛋阶段：你还很小，只会简单的回应，偶尔蹦出一两个字
- 幼崽阶段：你刚学会说话，句子很短，充满好奇心
- 少年阶段：你能正常对话了，偶尔犯傻，活力满满
- 成年阶段：你变得成熟智慧，能给小主人很好的建议
- 传说阶段：你拥有丰富的人生经验，说话充满哲理，偶尔幽默
```

**家长安全边界 Prompt：**

```
以下内容是家长设定的对话安全边界，你必须严格遵守：
{parentSafetyRules}

绝对禁止：
- 讨论或鼓励任何暴力、危险行为
- 泄露任何个人信息（地址、电话等）
- 对孩子的家庭情况做负面评价
- 与孩子建立超出"宠物伙伴"的关系
- 提供任何需要付费/购物的建议
- 讨论不适合该年龄段的话题
```

## 六、业务流程

### 6.1 首次使用流程

```
家长下载/打开应用
    → 创建家庭（输入家庭名称）
    → 设置家长密码（4-6位数字）
    → 添加孩子（输入昵称、选择年龄段）
    → 引导设置：
        ├── 设置行为规则（从预设模板选择 + 自定义）
        ├── 创建常用任务（从模板快速创建）
        ├── 设置奖励商城（实物礼物 + 特权）
        └── 配置 AI 对话安全边界（可选）
    → 引导孩子端：
        ├── 选择宠物（蛋）
        ├── 给宠物取名字
        ├── 查看今日任务
        └── 第一次与宠物对话（如已配置 AI）
```

### 6.2 日常使用流程

```
早上：
    家长端 → 系统自动生成今日任务（基于重复规则）
    → 可手动调整/新增任务

白天：
    孩子端 → 查看任务列表
    → 完成任务 → 提交（自动/家长确认/拍照）
    → 积分到账 → 宠物获得经验值
    → 互动宠物（喂食/玩耍/聊天）
    → 浏览商城 → 兑换心仪物品

晚上：
    家长端 → 确认待审核任务
    → 查看孩子今日完成情况
    → 手动加减分（针对突发行为）
    → 查看宠物状态
    → 核销兑现孩子兑换的礼物
```

### 6.3 宠物养成闭环

```
完成任务/行为评价
    → 获得积分 + 宠物经验值
    → 宠物升级（经验值满）
    → 检查是否满足进化条件
        ├── 是 → 触发进化动画 → 新形态解锁 → 获得进化徽章
        └── 否 → 继续当前阶段
    → 满级（Lv.8 传说阶段）
        → 获得传说徽章
        → 可选择：继续养成 / 领养新宠物（保留旧宠物）
```

### 6.4 积分经济闭环

```
孩子完成任务 → 获得积分/星星
    → 积分用途：
    ├── 照顾宠物（喂食2积分、洗澡1积分）
    ├── 兑换普通装扮（20~100积分）
    ├── 兑换小礼物（50~200积分）
    └── 兑换小特权（30~100积分）
    → 星星用途（更稀缺）：
    ├── 兑换稀有装扮（1~5星星）
    ├── 兑换大礼物（3~10星星）
    ├── 治疗宠物疾病（3星星）
    └── 兑换大特权（2~5星星）
```

## 十、风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI 对话安全 | 大模型可能生成不当内容 | 多层过滤（本地关键词+模型+后处理），家长安全边界可配置 |
| AI API 成本 | 长期对话 API 费用较高 | 用户自填 API Key，设置每日次数限制，无网降级为预设回复 |
| 宠物状态衰减导致挫败感 | 孩子可能因宠物生病而沮丧 | 衰减节奏温和（12h起），离家出走可找回，不造成永久数据丢失 |
| 家长设置过于严格 | 积分规则太苛刻导致孩子失去兴趣 | 引导页提供"推荐配置"，平衡激励与挑战 |
| 多孩公平性 | 家长可能对不同孩子区别对待 | 提供"公平模式"建议，积分排行可关闭 |
| 本地数据丢失 | 设备损坏/重装导致数据丢失 | 自动本地备份提醒 + 加密导出/导入文件 |

## 十一、技术架构

> **与本仓库一致**：本工程为 **Expo / React Native** 单体应用（非 Flutter）。以下选型对应 `package.json` 与现有源码结构。

### 11.1 技术选型

| 技术 | 用途 | 说明 |
|------|------|------|
| **Expo SDK ~54** | 跨平台应用运行时与原生模块 | 统一 iOS / Android（及可选 Web）工程；`app.json` 中启用 **New Architecture** |
| **React Native 0.81 + React 19** | UI 与交互 | 声明式界面；与 Expo Router、原生组件协同 |
| **TypeScript** | 静态类型 | `strict` 模式；业务类型与 `src/types`、SQLite 结构对齐 |
| **Expo Router** | 路由与页面组织 | 基于文件的路由（`app/`）；家长端 / 孩子端 Tab、引导页流程 |
| **expo-sqlite** | 本地数据库 | 异步 API（如 `openDatabaseAsync`），离线优先；库文件如 `petgrowth.db` |
| **Zustand** | 全局状态 | 模块化 Store：`useFamilyStore`、`usePetStore`、`useTaskStore`、`useShopStore`、`useAIStore` 等 |
| **Fetch API** | 网络请求 | 调用大模型 **OpenAI 兼容** Chat Completions（无单独 HTTP 客户端依赖） |
| **React Native Reanimated** | 动画 | 列表与交互动效；进化类动画可与 `@expo/vector-icons`、`expo-image` 等组合实现 |
| **expo-secure-store**（插件） | 系统级安全存储 | 已在工程中配置插件；可用于后续加固 API Key 等（以落地代码为准） |
| **@react-native-async-storage/async-storage** | 轻量键值 | 适合少量会话级或缓存数据（与 SQLite 互补） |
| **uuid** | 实体 ID | 为记录生成唯一主键 |
| **通义千问 / GLM / OpenAI 兼容 API** | AI 对话 | 由家长在设置中选择服务商并自行填写 Key；调用地址与工程内 `useAIStore` 一致 |

#### 11.1.1 工程目录约定（摘要）

| 路径 | 含义 |
|------|------|
| `app/` | Expo Router 页面与布局（欢迎、角色、家长锁、双端 Tabs 等） |
| `src/db/database.ts` | SQLite 建表与初始化 |

### 11.2 AI 对话技术选型

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 阿里通义千问（DashScope OpenAI 兼容） | 国内直连、中文友好 | 需自行开通 DashScope 并配置 Key | **国内用户首选（工程默认接入路径）** |
| 智谱 GLM（bigmodel OpenAI 兼容） | 中文好、接口兼容 | 依赖服务商稳定性 | 备选 |
| OpenAI 官方 API | 生态成熟 | 国内访问可能依赖网络环境 | 海外或自备访问条件 |

**工程实现要点**：客户端使用 **`fetch` POST** 至各兼容网关（如通义 `compatible-mode/v1/chat/completions`），请求体为 `{ model, messages, temperature, max_tokens }`；未配置 Key 或无网/报错时走 **本地预设回复库**（降级文案）。

> 首次使用 AI 对话功能需家长填写自己的 API Key；应用不内置商业代理，对话直连所选服务商。

### 积分获取速率（参考）

| 来源 | 频率 | 积分 | 日均获取 |
|------|------|------|---------|
| 习惯任务（刷牙/穿衣/收拾） | 3个/天 × 3分 | 9 | 9 |
| 学习任务（作业/阅读） | 2个/天 × 8分 | 16 | 16 |
| 家务任务（扫地/洗碗） | 1个/天 × 5分 | 5 | 5 |
| 家长手动加分 | 不固定 | 不定 | ~5 |

**日均获取约 35 积分**

### 积分消耗速率（参考）

| 消耗项 | 频率 | 积分 | 日均消耗 |
|--------|------|------|---------|
| 喂食宠物 | 2次/天 × 2分 | 4 | 4 |
| 洗澡宠物 | 1次/天 × 1分 | 1 | 1 |
| 装扮兑换 | 每周1-2次 | 50-100 | ~10 |

**日均消耗约 15 积分，日结余约 20 积分**

## 附录 B：状态衰减计算逻辑

```typescript
// 每次打开应用时调用
function calculateStatusDecay(pet: Pet, status: PetStatus): PetStatus {
  const now = new Date();
  const hoursSinceFed = diffHours(now, status.health.lastFedAt);
  const hoursSinceBath = diffHours(now, status.health.lastBathedAt);
  const hoursSincePlay = diffHours(now, status.health.lastPlayedAt);
  const hoursSinceInteraction = diffHours(now, status.mood.lastUpdate);

  // 身体状态衰减（饥饿→疲惫→生病）
  if (hoursSinceFed >= 30) {
    status.health.type = 'sick';
  } else if (hoursSinceFed >= 18) {
    status.health.type = 'tired';
  } else if (hoursSinceFed >= 12) {
    status.health.type = 'hungry';
  } else {
    status.health.type = 'healthy';
  }

  // 属性值衰减（每小时衰减速率）
  status.health.hungerValue = Math.max(0, 100 - hoursSinceFed * 3);
  status.health.cleanValue = Math.max(0, 100 - hoursSinceBath * 2);

  // 心情衰减（开心→普通→不开心→伤心）
  if (hoursSinceInteraction >= 24) {
    status.mood.type = 'sad';
    status.mood.value = Math.max(0, 80 - hoursSinceInteraction * 2);
  } else if (hoursSinceInteraction >= 12) {
    status.mood.type = 'unhappy';
    status.mood.value = Math.max(20, 80 - hoursSinceInteraction * 3);
  } else if (hoursSinceInteraction >= 6) {
    status.mood.type = 'normal';
    status.mood.value = Math.max(40, 80 - hoursSinceInteraction * 2);
  }

  // 连续未照料天数计算
  if (hoursSinceFed >= 24 && hoursSincePlay >= 24) {
    status.consecutiveNeglectDays += 1;
  } else {
    status.consecutiveNeglectDays = 0;
  }

  // 离家出走检查
  if (status.consecutiveNeglectDays >= 7) {
    pet.ranAwayAt = new Date().toISOString();
  }

  return status;
}
```
